# Пояснення коду: Task 05 (Черга через лінійний зв'язаний список)

Оскільки черга вимагає додавання елементів з одного кінця і вилучення з іншого (FIFO), нам недостатньо мати доступ лише до початку (`_head`) як у стеку. Щоб досягти часу О(1) для всіх операцій, ми додаємо додатковий вказівник на кінець черги — `_tail`.

## Розбір класу:

```csharp
private Node<T>? _head; // вказує на найстаріший (перший) елемент
private Node<T>? _tail; // вказує на найновіший (останній) елемент
```
Поля нашої черги. Коли ми щось дістаємо — беремо з `_head`. Коли хочемо додати — чіпляємо після `_tail`.

```csharp
public void Enqueue(T item)
{
    Node<T> newNode = new Node<T>(item);
    
    if (_tail == null)
    {
        _head = newNode;
        _tail = newNode;
    }
    else
    {
        _tail.Next = newNode;
        _tail = newNode;
    }
    _count++;
}
```
**Додавання:**
Якщо черга порожня (`_tail == null`), то новий вузол стає одночасно і головою, і хвостом черги — адже він там єдиний.
Якщо черга не порожня, ми просто беремо поточний хвіст і його властивістю `Next` "причіпляємо" туди новий вузол `_tail.Next = newNode`. Після цього оновлюємо саму змінну хвоста, що тепер наш хвіст — це останній доданий вузол. Все відбувається миттєво, **O(1)**.

```csharp
public T Dequeue()
{
    T value = _head!.Data;
    _head = _head.Next;
    
    if (_head == null)
    {
        _tail = null;
    }
    
    _count--;
    return value;
}
```
**Вилучення:**
Вилучення ідентичне до стеку. Ми беремо значення з голови, і пересуваємо голову на наступний елемент `_head = _head.Next`. 
Але є один нюанс: якщо ми вилучили останній елемент, змінна `_head` стане рівною вихідно-пустому `null`. Але наша змінна `_tail` тим часом все ще посилається на недійсний видалений об'єкт. Тому ми додаємо перевірку `if (_head == null) _tail = null;`, щоб повністю очистити чергу. **O(1)**.
