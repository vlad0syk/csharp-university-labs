# Пояснення коду: Task 04 (Стек через лінійний зв'язаний список)

Завдяки зв'язаним спискам нам більше не потрібен фіксований масив. Наш елемент стає незалежним об'єктом, який знає, де лежить наступний елемент. Стек працює за принципом LIFO, тому ми завжди додаємо і удаляємо елементи лише з початку ("Голови").

## Розбір класів:

```csharp
public class Node<T>
{
    public T Data { get; set; }
    public Node<T>? Next { get; set; }
}
```
Це клас-структура вузла (Node). В ньому лежить саме значення `Data` і посилання (вказівник) на наступний об'єкт нашого списку `Next`.

```csharp
private Node<T>? _head;
```
Поле класу `LinkedListStack`. `_head` (голова) — це фактично вершина нашого стеку. Все що нижче голови — зв'язано ланцюжком через властивість `Next`.

```csharp
public void Push(T item)
{
    Node<T> newNode = new Node<T>(item);
    newNode.Next = _head;
    _head = newNode;
    _count++;
}
```
Додавання до стеку:
1. Створюємо новий вузол з нашим числом.
2. Кажемо йому: "Твій сусід знизу — це поточна `_head`". (`newNode.Next = _head`). 
3. Кажемо стеку: "Тепер ти починаєшся з цього нового вузла" (`_head = newNode`).
Виконується за **O(1)**.

```csharp
public T Pop()
{
    T value = _head!.Data;
    _head = _head.Next;
    _count--;
    return value;
}
```
Видалення зі стеку:
1. Забираємо дані поточної `_head`.
2. Відрізаємо голову від стеку, сказавши, що тепер "голова" — це наступний після неї елемент (`_head = _head.Next`). Попередній об'єкт просто зависає в пам'яті поки не очиститься автоматичним Garbage Collector-ом C#.
Виконується за **O(1)**.
