## Завдання 1: Ілюстрація PARTITION

> **Умова:** Проілюструйте роботу процедури PARTITION з масивом A = (13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11).

### Алгоритм

PARTITION розділяє масив на дві частини відносно опорного елемента (pivot). Pivot — **останній елемент** масиву.

- Два покажчики: `left` (межа елементів ≤ pivot) і `j` (сканує масив зліва направо)
- Якщо `A[j] ≤ pivot` → збільшуємо `left`, міняємо `A[left]` та `A[j]`
- В кінці ставимо pivot на позицію `left + 1`

### Покрокове виконання

**Вхід:** `[13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11]`, pivot = **11**

| Крок | j | Дія | Стан масиву |
|------|---|-----|-------------|
| 0 | — | Виділяємо pivot = 11 (зелений) | `[13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, \|11\|]` |
| 1 | 0 | 13 > 11 → пропускаємо | `[13, 19, 9, 5, 12, 8, 7, 4, 21, 2, 6, 11]` |
| 2 | 1 | 19 > 11 → пропускаємо | без змін |
| 3 | 2 | 9 ≤ 11 → swap, left=0 | `[9, 19, 13, 5, 12, 8, 7, 4, 21, 2, 6, 11]` |
| 4 | 3 | 5 ≤ 11 → swap, left=1 | `[9, 5, 13, 19, 12, 8, 7, 4, 21, 2, 6, 11]` |
| 5 | 4 | 12 > 11 → пропускаємо | без змін |
| 6 | 5 | 8 ≤ 11 → swap, left=2 | `[9, 5, 8, 19, 12, 13, 7, 4, 21, 2, 6, 11]` |
| 7 | 6 | 7 ≤ 11 → swap, left=3 | `[9, 5, 8, 7, 12, 13, 19, 4, 21, 2, 6, 11]` |
| 8 | 7 | 4 ≤ 11 → swap, left=4 | `[9, 5, 8, 7, 4, 13, 19, 12, 21, 2, 6, 11]` |
| 9 | 8 | 21 > 11 → пропускаємо | без змін |
| 10 | 9 | 2 ≤ 11 → swap, left=5 | `[9, 5, 8, 7, 4, 2, 19, 12, 21, 13, 6, 11]` |
| 11 | 10 | 6 ≤ 11 → swap, left=6 | `[9, 5, 8, 7, 4, 2, 6, 12, 21, 13, 19, 11]` |
| **Фінал** | — | Swap pivot на left+1 | `[9, 5, 8, 7, 4, 2, 6, \|11\|, 21, 13, 19, 12]` |

### Результат

```
Ліва частина (≤ 11): [9, 5, 8, 7, 4, 2, 6]
Pivot:               11  (індекс q = 7)
Права частина (> 11): [21, 13, 19, 12]
```

### Код

```csharp
private static int PartitionDescending(int[] array, int p, int r)
{
    int pivot = array[r];       // Опорний = останній елемент
    int i = p - 1;              // Межа "лівої" частини

    for (int j = p; j <= r - 1; j++)
    {
        if (array[j] >= pivot)  // Для спадання: >=, для зростання: <=
        {
            i++;
            Swap(array, i, j);
        }
    }

    Swap(array, i + 1, r);     // Ставимо pivot на своє місце
    return i + 1;              // Повертаємо індекс pivot
}
```

**Складність:** O(n) — один прохід по масиву.

## Завдання 5: Ілюстрація COUNTING-SORT

> **Умова:** Проілюструйте обробку масиву A = (6, 0, 2, 0, 1, 3, 4, 6, 1, 3, 2) процедурою COUNTING-SORT.

### Алгоритм

COUNTING-SORT — **не порівняльне** сортування. Підраховує кількість кожного значення, потім розміщує елементи на правильні позиції.

### Покрокове виконання

**Крок 1: Створюємо масив підрахунку (count)**

Для кожного значення `0..6` рахуємо, скільки разів воно зустрічається:

```
Вхід:    [6, 0, 2, 0, 1, 3, 4, 6, 1, 3, 2]

Індекс:   0  1  2  3  4  5  6
Count:  [ 2, 2, 2, 2, 1, 0, 2 ]
          ↑  ↑  ↑  ↑  ↑  ↑  ↑
          0  1  2  3  4  5  6
        з'яв. з'яв. ...
```

- `0` зустрічається **2** рази
- `1` зустрічається **2** рази
- `2` зустрічається **2** рази
- `3` зустрічається **2** рази
- `4` зустрічається **1** раз
- `5` зустрічається **0** разів
- `6` зустрічається **2** рази

**Крок 2: Обчислюємо накопичувальні суми (prefix sum)**

Кожен елемент стає сумою всіх попередніх:

```
Count:    [ 2, 2, 2, 2, 1, 0, 2 ]
                  ↓
Prefix:   [ 2, 4, 6, 8, 9, 9, 11 ]
```

- `prefix[i]` вказує, скільки елементів ≤ `i` — тобто де закінчується блок значення `i` у вихідному масиві

**Крок 3: Будуємо вихідний масив (справа наліво для стабільності)**

Проходимо вхідний масив **з кінця**, ставимо кожен елемент на позицію `prefix[value] - 1`, зменшуємо `prefix[value]`:

```
Вхід (зправа наліво):  2, 3, 1, 6, 4, 3, 1, 0, 2, 0, 6

Вихід: [ 0, 0, 1, 1, 2, 2, 3, 3, 4, 6, 6 ]
```

### Код

```csharp
private static int[] CountingSort(int[] array, int k)
{
    int[] counts = BuildCounts(array, k);       // Крок 1
    int[] prefix = BuildPrefixCounts(counts);    // Крок 2
    int[] output = new int[array.Length];

    for (int j = array.Length - 1; j >= 0; j--)  // Крок 3 (справа наліво)
    {
        int value = array[j];
        output[prefix[value] - 1] = value;
        prefix[value]--;
    }

    return output;
}
```

**Складність:** O(n + k), де n — розмір масиву, k — максимальне значення. Сортування **стабільне** — зберігає порядок однакових елементів.

## Завдання 7: Ілюстрація RADIX-SORT

> **Умова:** Проілюструйте роботу алгоритму RADIX-SORT з наступним списком англійських слів: COW, DOG, SEA, RUG, ROW, MOB, BOX, TAB, BAR, EAR, TAR, DIG, BIG, TEA, NOW, FOX.

### Алгоритм

RADIX-SORT сортує **посимвольно**, починаючи від **останнього** символу до першого. На кожному кроці використовує **стабільне** сортування (Counting Sort по символу).

### Покрокове виконання

Всі слова мають 3 літери. Сортуємо від 3-ї позиції до 1-ї.

**Прохід 0 (вхід):**
```
COW  DOG  SEA  RUG  ROW  MOB  BOX  TAB  BAR  EAR  TAR  DIG  BIG  TEA  NOW  FOX
```

**Прохід 1 — сортуємо за 3-м символом** (остання літера):

```
SEA  TEA  TAB  MOB  DOG  RUG  DIG  BIG  BAR  EAR  TAR  COW  ROW  NOW  BOX  FOX
 A    A    B    B    G    G    G    G    R    R    R    W    W    W    X    X
```

**Прохід 2 — сортуємо за 2-м символом** (зберігаючи порядок з проходу 1):

```
TAB  BAR  EAR  TAR  SEA  TEA  DIG  BIG  MOB  DOG  COW  ROW  NOW  BOX  FOX  RUG
 A    A    A    A    E    E    I    I    O    O    O    O    O    O    O    U
```

**Прохід 3 — сортуємо за 1-м символом** (фінальний результат):

```
BAR  BIG  BOX  COW  DIG  DOG  EAR  FOX  MOB  NOW  ROW  RUG  SEA  TAB  TAR  TEA
 B    B    B    C    D    D    E    F    M    N    R    R    S    T    T    T
```

### Чому потрібна стабільність?

Після сортування за 3-м символом слова вже впорядковані за останньою літерою. Коли сортуємо за 2-м символом, слова з **однаковою** 2-ю літерою повинні **зберегти** порядок за 3-ю. Це гарантує лише **стабільне** сортування.

### Код

```csharp
private static void RadixSortWords(string[] words)
{
    int maxLength = words.Max(w => w.Length);
    for (int pos = maxLength - 1; pos >= 0; pos--)  // Від останнього символу до першого
    {
        CountingSortByChar(words, pos);              // Стабільне сортування за pos-символом
    }
}
```

**Складність:** O(d · (n + k)), де d — довжина слова, n — кількість слів, k — розмір алфавіту.

## Завдання 8: Ілюстрація BUCKET-SORT

> **Умова:** Проілюструйте роботу алгоритму BUCKET-SORT з масивом A = (.79, .13, .16, .64, .39, .20, .89, .53, .71, .42).

### Алгоритм

BUCKET-SORT працює з числами з діапазону `[0, 1)`. Розподіляє елементи в «кошики» (buckets), сортує кожен кошик окремо, потім зливає.

### Покрокове виконання

**Крок 1: Розподіл по кошиках**

Номер кошика = `⌊value × n⌋`, де n = 10 (кількість елементів).

```
Вхід:  [.79, .13, .16, .64, .39, .20, .89, .53, .71, .42]

Кошик 0: (пусто)
Кошик 1: .13, .16
Кошик 2: .20
Кошик 3: .39
Кошик 4: .42
Кошик 5: .53
Кошик 6: .64
Кошик 7: .79, .71
Кошик 8: .89
Кошик 9: (пусто)
```

**Крок 2: Сортування кожного кошика**

```
Кошик 1: .13, .16  (вже відсортований)
Кошик 7: .71, .79  (було .79, .71 → відсортували)
```

Усі інші кошики мають ≤1 елемент, тому вже відсортовані.

**Крок 3: Злиття кошиків послідовно**

```
Вихід: [.13, .16, .20, .39, .42, .53, .64, .71, .79, .89]
```

### Код

```csharp
private static double[] BucketSort(double[] array)
{
    int n = array.Length;
    List<double>[] buckets = new List<double>[n];
    for (int i = 0; i < n; i++)
        buckets[i] = new List<double>();

    foreach (double value in array)                     // Крок 1: розподіл
    {
        int bucketIndex = Math.Min(n - 1, (int)(value * n));
        buckets[bucketIndex].Add(value);
    }

    for (int i = 0; i < n; i++)
        buckets[i].Sort();                              // Крок 2: сортування кошиків

    double[] result = new double[n];
    int idx = 0;
    for (int i = 0; i < n; i++)                         // Крок 3: злиття
        foreach (double value in buckets[i])
            result[idx++] = value;

    return result;
}
```

**Складність:** O(n) в середньому (при рівномірному розподілі), O(n²) у найгіршому випадку.

## Завдання 9: Оптимальне розташування трубопроводу

> **Умова:** Знайти горизонтальний рівень трубопроводу `y*`, який мінімізує сумарну довжину вертикальних рукавів до свердловин. Показати, що це можна зробити за лінійний час.

### Математичне обґрунтування

Мінімізуємо `S(y*) = Σ|yᵢ - y*|`. Ця сума мінімальна при `y* = median(y₁, ..., yₙ)`.

> **Чому медіана?** При зсуві від медіани кількість свердловин «за» та «проти» зсуву однакова — зміни компенсуються, і будь-який зсув лише збільшує суму.

### Покрокове виконання

**Вхід:** 7 свердловин: `(2,9), (4,1), (8,6), (10,3), (13,8), (14,4), (18,7)`

**Крок 1:** Витягуємо y-координати: `[9, 1, 6, 3, 8, 4, 7]`

**Крок 2:** Знаходимо медіану через **Median of Medians** (детермінований O(n)):
- Групи по 5: `[9,1,6,3,8]` → медіана 6; `[4,7]` → медіана 7
- Медіана медіан → pivot = 6
- Розподіл: lows=`[1,3,4]`, pivots=`[6]`, highs=`[9,8,7]`
- k=3, lows.Count=3 → потрапляє в pivots → **медіана = 6**

**Крок 3:** Рахуємо суму: `|9-6|+|1-6|+|6-6|+|3-6|+|8-6|+|4-6|+|7-6| = 3+5+0+3+2+2+1 = 16`

### Результат

```
Optimal pipeline level y*: 6
Total branch length:       16
```

### Алгоритм Median of Medians

```csharp
private static int SelectKthDeterministic(int[] array, int k)
{
    if (array.Length <= 5)           // Базовий випадок
    {
        Array.Sort(array);
        return array[k];
    }

    // Розбиваємо на групи по 5, знаходимо медіану кожної
    List<int> medians = new List<int>();
    for (int i = 0; i < array.Length; i += 5)
    {
        int length = Math.Min(5, array.Length - i);
        int[] group = new int[length];
        Array.Copy(array, i, group, 0, length);
        Array.Sort(group);
        medians.Add(group[length / 2]);
    }

    // Рекурсивно знаходимо медіану медіан
    int pivot = SelectKthDeterministic(medians.ToArray(), medians.Count / 2);

    // Розподіляємо на три частини
    List<int> lows = new(), highs = new(), pivots = new();
    foreach (int v in array)
    {
        if (v < pivot) lows.Add(v);
        else if (v > pivot) highs.Add(v);
        else pivots.Add(v);
    }

    if (k < lows.Count) return SelectKthDeterministic(lows.ToArray(), k);
    if (k < lows.Count + pivots.Count) return pivot;
    return SelectKthDeterministic(highs.ToArray(), k - lows.Count - pivots.Count);
}
```

**Складність:** O(n) — **гарантовано** у найгіршому випадку (на відміну від QuickSelect, який O(n²) у найгіршому).

## 📊 Порівняння складностей алгоритмів

| Алгоритм | Середній | Найгірший | Стабільний | Тип |
|----------|----------|-----------|------------|-----|
| PARTITION | O(n) | O(n) | — | Допоміжний |
| COUNTING-SORT | O(n+k) | O(n+k) | ✅ Так | Не порівняльний |
| RADIX-SORT | O(d(n+k)) | O(d(n+k)) | ✅ Так | Не порівняльний |
| BUCKET-SORT | O(n) | O(n²) | ✅ Так | Не порівняльний |
| Median of Medians | O(n) | O(n) | ❌ Ні | Вибір k-го |
