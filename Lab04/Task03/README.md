# Пояснення коду: Task 03 (Стек за допомогою 2-х черг)

Цей код навпаки: моделює роботу **стеку** (останній прийшов — перший вийшов) за допомогою двох **черг** (перший прийшов — перший вийшов). Щоб зробити читання стеку миттєвим, ми одразу обробляємо дані ще на моменті їх вставки (Push) так, щоб останній доданий елемент завжди опинявся на початку першої черги.

## Розбір класу `StackTwoQueues`:

```csharp
private Queue<T> _q1;
private Queue<T> _q2;
```
Маємо дві черги. `_q1` завжди буде головною чергою (де лежать всі елементи). `_q2` буде використовуватись тимчасово як допоміжна черга для перестановки елементів місцями.

```csharp
public void Push(T item)
{
    _q2.Enqueue(item);
    while (_q1.Count > 0)
    {
        _q2.Enqueue(_q1.Dequeue());
    }
    (_q1, _q2) = (_q2, _q1);
}
```
Це найголовніший метод усього класу (працює за **O(N)**):
1. Коли ми додаємо новий елемент, ми спочатку кладемо його у порожню допоміжну чергу `_q2`. (Наприклад, ми додали "3", тепер `_q2 = [3]`).
2. Далі ми беремо всі елементи з основної черги `_q1` (наприклад, там лежать `[2, 1]`) і по черзі перекладаємо їх у `_q2` (після перекладання `_q2 = [3, 2, 1]`). 
3. Наостанок міняємо змінні місцями (свап), тобто робимо так, щоб `_q1` тепер було `[3, 2, 1]`, а `_q2` стала назад порожньою `[]`.
Як бачите, тепер наш найновіший елемент (3) лежить на самому початку.

```csharp
public T Pop()
{
    if (IsEmpty) throw new InvalidOperationException("Stack is empty");
    return _q1.Dequeue();
}
```
Оскільки ми всю важку роботу (імітацію LIFO) зробили в методі `Push`, тут нам достатньо просто виконати стандартний метод черги `Dequeue()`, який гарантовано видасть "найновіший" елемент, бо ми його завчасно поставили на початок. Складність **O(1)**.
